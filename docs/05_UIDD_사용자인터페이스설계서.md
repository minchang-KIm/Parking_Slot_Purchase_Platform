## 1. 문서 개요

### 1.1 목적
본 문서는 주차 거래 플랫폼의 사용자 인터페이스 설계를 상세히 기술한다. UI/UX 디자이너와 프론트엔드 개발자가 일관된 사용자 경험을 제공하기 위한 가이드라인을 제시한다.

### 1.2 범위
- 화면 설계 및 레이아웃
- 사용자 플로우
- 컴포넌트 설계
- 디자인 시스템
- 반응형 디자인

---

## 2. 디자인 원칙

### 2.1 핵심 원칙

#### 2.1.1 단순성 (Simplicity)
- 복잡한 기능을 단순하고 직관적인 인터페이스로 제공
- 불필요한 요소 제거, 핵심 기능에 집중

#### 2.1.2 일관성 (Consistency)
- 전체 애플리케이션에서 일관된 디자인 언어 사용
- 동일한 요소는 동일한 모습과 동작

#### 2.1.3 접근성 (Accessibility)
- 모든 사용자가 쉽게 접근하고 사용할 수 있는 디자인
- WCAG 2.1 가이드라인 준수

#### 2.1.4 반응성 (Responsiveness)
- 다양한 화면 크기에서 최적화된 경험 제공
- 모바일 우선 접근 방식

## 4. 컴포넌트 설계

### 4.1 버튼 (Buttons)

#### Primary Button
```css
/* 주요 액션 버튼 */
.btn-primary {
  background-color: var(--primary-600);
  color: white;
  padding: 0.5rem 1rem;
  border-radius: var(--radius-lg);
  font-weight: var(--font-medium);
  transition: all 0.2s;
}

.btn-primary:hover {
  background-color: var(--primary-700);
}

.btn-primary:focus {
  outline: 2px solid var(--primary-500);
  outline-offset: 2px;
}
```

**사용 예시**:
- 회원가입, 로그인
- 주차공간 등록
- 예약하기, 결제하기

```css
/* 보조 액션 버튼 */

**사용 예시**:

## 6. 사용자 플로우

### 6.1 주차공간 예약 플로우

```
[홈페이지]
    ↓ 검색 버튼 클릭
[검색 페이지]
    ↓ 필터 적용 및 검색
[검색 결과]
    ↓ 주차공간 선택
[상세 페이지]
    ↓ 예약 정보 입력
    ↓ 예약하기 버튼
[로그인 확인]
    ↓ (로그인 안됨) → [로그인 페이지] → 로그인 후 복귀
    ↓ (로그인 됨)
[예약 확인]
    ↓ 확인 버튼
[결제 페이지]
    ↓ 결제 정보 입력
    ↓ 결제하기
[결제 처리]
    ↓ (성공)
[예약 완료]
    ↓ 토스트: "예약이 완료되었습니다"
[내 예약 페이지]
```

### 6.2 주차공간 등록 플로우

```
[내 주차공간 페이지]
    ↓ '새 공간 등록' 버튼
[등록 폼]
    ↓ 기본 정보 입력
    ↓ 위치 정보 입력 (지도 선택)
    ↓ 가격 정보 입력
    ↓ 이미지 업로드
    ↓ 등록 버튼
[유효성 검증]
    ↓ (성공)
[등록 완료]
    ↓ 토스트: "주차공간이 등록되었습니다"
[내 주차공간 페이지]
```

### 6.3 리뷰 작성 플로우

```
[내 예약 페이지]
    ↓ 완료된 예약 선택
    ↓ '리뷰 작성' 버튼
[리뷰 작성 폼]
    ↓ 평점 선택 (1-5)
    ↓ 리뷰 내용 작성
    ↓ (선택) 이미지 업로드
    ↓ 작성 버튼
[리뷰 제출]
    ↓ 토스트: "리뷰가 작성되었습니다"
[주차공간 상세 페이지]
    ↓ 작성된 리뷰 표시
```

  /* 터치 친화적 버튼 크기 */
}

/* 태블릿 */
@media (min-width: 640px) and (max-width: 1023px) {
  /* 2열 레이아웃 */
  /* 사이드바 접기/펼치기 */
}

/* 데스크톱 */
@media (min-width: 1024px) {
  /* 3열 레이아웃 */
  /* 사이드바 항상 표시 */
  /* 호버 효과 */
}
```

## 9. 접근성 (Accessibility)

### 9.1 키보드 내비게이션
- Tab 키로 모든 인터랙티브 요소 접근 가능

- aria-label 사용

- WCAG AA 준수: 최소 4.5:1
- 색상만으로 정보 전달하지 않음
